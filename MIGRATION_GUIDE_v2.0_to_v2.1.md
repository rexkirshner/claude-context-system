# Migration Guide: v2.0 → v2.1

**Target Audience:** Projects currently using Claude Context System v2.0
**Migration Time:** 10-15 minutes
**Difficulty:** Easy (mostly automatic)

---

## What's New in v2.1

### File Consolidation
- **QUICK_REF.md removed** → Merged into STATUS.md as auto-generated section
- Reduces file count: 6 → 5 files (4 core + 1 AI header)
- Eliminates manual drift between QUICK_REF and STATUS

### Platform Neutrality
- **AI header pattern introduced** (claude.md, cursor.md, aider.md, codex.md)
- 7-line redirects to platform-neutral CONTEXT.md
- Easy multi-AI tool support

### Enhanced Features
- **Mandatory TL;DR** in SESSIONS.md (enforced in template)
- **Auto-logged git operations** (approval tracking)
- **Automated staleness detection** (configurable thresholds)
- **CODE_MAP.md optional** (4-question adoption criteria)
- **5-layer git push protection** (structural prevention)

### Quality of Life
- CONTEXT.md trimmed: 600 → 282 lines (50% reduction)
- /update-templates command for easy template updates
- /add-ai-header command for multi-tool teams
- /session-summary command for quick history navigation

## Breaking Changes

### QUICK_REF.md No Longer Exists

**What changed:**
- v2.0: Separate `context/QUICK_REF.md` file
- v2.1: Quick Reference section within `context/STATUS.md` (auto-generated)

**Why:**
- Eliminates duplication
- Single source of truth
- Auto-generation prevents manual drift

**Migration required:** Yes (automatic with migration steps below)

## Migration Steps

### Option 1: Automatic Migration (Recommended)

**Prerequisites:**
1. All changes committed to git
2. Working directory clean
3. Backup created (optional but recommended)

**Steps:**

```bash
# 1. Update the context system
/update-context-system

# 2. Migrate QUICK_REF.md → STATUS.md
# This command will:
# - Move QUICK_REF content to top of STATUS.md
# - Delete QUICK_REF.md
# - Update references in other files
# - Add AI header (claude.md)

# Run in terminal:
cd /path/to/your/project

# Backup (optional)
cp -r context context.backup

# Create STATUS.md Quick Reference section from QUICK_REF.md
if [ -f "context/QUICK_REF.md" ]; then
  echo "## 📊 Quick Reference" > /tmp/quick-ref-section.md
  echo "_(This section is auto-generated by /save and /save-full - DO NOT edit manually)_" >> /tmp/quick-ref-section.md
  echo "" >> /tmp/quick-ref-section.md
  cat context/QUICK_REF.md >> /tmp/quick-ref-section.md
  echo "" >> /tmp/quick-ref-section.md
  echo "---" >> /tmp/quick-ref-section.md
  echo "" >> /tmp/quick-ref-section.md

  # Insert at top of STATUS.md (after front matter if exists)
  if grep -q "^---" context/STATUS.md 2>/dev/null; then
    # Has front matter, insert after second ---
    awk '/^---/ { count++; if (count == 2) { print; system("cat /tmp/quick-ref-section.md"); next }} 1' \
      context/STATUS.md > context/STATUS.md.tmp
    mv context/STATUS.md.tmp context/STATUS.md
  else
    # No front matter, insert at top
    cat /tmp/quick-ref-section.md context/STATUS.md > context/STATUS.md.tmp
    mv context/STATUS.md.tmp context/STATUS.md
  fi

  rm /tmp/quick-ref-section.md
  echo "✅ Merged QUICK_REF.md into STATUS.md"

  # Delete old QUICK_REF.md
  rm context/QUICK_REF.md
  echo "✅ Deleted context/QUICK_REF.md"
else
  echo "⚠️  context/QUICK_REF.md not found (already migrated?)"
fi

# 3. Create AI header (claude.md)
if [ ! -f "context/claude.md" ]; then
  curl -sL https://raw.githubusercontent.com/rexkirshner/claude-context-system/main/templates/claude.md.template \
    -o context/claude.md
  echo "✅ Created context/claude.md"
else
  echo "ℹ️  context/claude.md already exists"
fi

# 4. Update .context-config.json to v2.1.0
# Download latest template
curl -sL https://raw.githubusercontent.com/rexkirshner/claude-context-system/main/config/.context-config.template.json \
  -o context/.context-config.json.new

# Update version field (preserving your customizations)
jq -s '.[0] * .[1]' context/.context-config.json context/.context-config.json.new > context/.context-config.json.merged
mv context/.context-config.json.merged context/.context-config.json
rm context/.context-config.json.new
echo "✅ Updated .context-config.json to v2.1.0"

# 5. Run /validate-context to verify
echo ""
echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"
echo "✅ Migration Complete!"
echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"
echo ""
echo "Next steps:"
echo "1. Review context/STATUS.md - Quick Reference section at top"
echo "2. Run: /validate-context"
echo "3. Run: /save to regenerate Quick Reference with latest data"
echo "4. Commit migration: git add context/ && git commit -m 'Migrate to v2.1.0'"
```

### Option 2: Manual Migration

If you prefer manual control:

**Step 1: Merge QUICK_REF into STATUS**

1. Open `context/QUICK_REF.md`
2. Copy entire contents
3. Open `context/STATUS.md`
4. Add this at the top (after any front matter):

```markdown
## 📊 Quick Reference
_(This section is auto-generated by /save and /save-full - DO NOT edit manually)_

[Paste QUICK_REF.md content here]

---
```

5. Delete `context/QUICK_REF.md`

**Step 2: Create AI Header**

1. Create `context/claude.md` with:

```markdown
# Claude Context

**📍 Start here:** [CONTEXT.md](./CONTEXT.md)

This project uses the Claude Context System v2.1. All documentation is in platform-neutral markdown files.

**Quick start:** [STATUS.md](./STATUS.md) → Active Tasks → Begin working
```

**Step 3: Update Config**

1. Open `context/.context-config.json`
2. Change `"version": "2.0.0"` → `"version": "2.1.0"`
3. Add new fields from [template](https://raw.githubusercontent.com/rexkirshner/claude-context-system/main/config/.context-config.template.json):
   - `project.urls.*`
   - `project.commands.*`
   - `validation.stalenessThresholds`
   - `git.pushProtection`
   - `documentation.aiHeaders`

**Step 4: Verify**

1. Run `/validate-context`
2. Run `/save` to regenerate Quick Reference
3. Check `context/STATUS.md` has Quick Reference section at top

## Troubleshooting

### Quick Reference not auto-generating

**Problem:** STATUS.md Quick Reference section not updating when running /save

**Solution:**
1. Ensure section header exists: `## 📊 Quick Reference`
2. Check next line has: `_(This section is auto-generated by /save and /save-full - DO NOT edit manually)_`
3. Run `/save` again
4. Verify `.context-config.json` has `project.name`, `project.urls`, etc.

### QUICK_REF.md still referenced

**Problem:** Old commands still mention QUICK_REF.md

**Solution:**
```bash
/update-context-system  # Updates all commands to v2.1
```

### Lost custom Quick Reference content

**Problem:** Had custom sections in QUICK_REF.md

**Solution:**
1. Check backup: `context.backup/QUICK_REF.md`
2. Add custom content to appropriate sections:
   - Project info → `context/.context-config.json` (auto-populates Quick Reference)
   - Quick links → `context/CONTEXT.md` Key Resources section
   - Dashboard data → STATUS.md sections (auto-summarizes in Quick Reference)

### Staleness showing wrong dates

**Problem:** All files show as stale immediately

**Solution:**
1. Check system clock is correct
2. Verify `.context-config.json` thresholds are reasonable
3. Run `/save` to update STATUS.md timestamp
4. Run `/save-full` to update SESSIONS.md
